Generated SQL 
select
  le.assessment_reference,
  le.property_address,
  le.street,
  le.postcode,
  smv.total_area,
  smv.rateable_value,
  round(smv.rateable_value / nullif(smv.total_area, 0), 2) as rv_per_sqm,
  round(percentile_cont(0.5) within group (
    order by smv.rateable_value / nullif(smv.total_area, 0)
  ) over (partition by le.street), 2) as street_median
from list_entries le
join smv_assessments smv using (assessment_reference)
where smv.total_area > 0
order by (smv.rateable_value / nullif(smv.total_area, 0)) desc
limit 8;

What the AI checks

Every answer is grounded in the rating-list tables. The SQL is exposed so the working can be inspected.

  • Comparable context Neighbours, parade peers, postcode and scheme references.
  • £/sqm pressure Outliers against street and scheme medians.
  • Zoning evidence Zone A / B / C breakdowns from summary valuation line items.

What this is not

  • Not an appeal recommendation Anomalies are leads for surveyor review.
  • Not rental evidence The platform sees the list, not the market.
  • Not measurement Floor areas come from VOA records, not site survey.
